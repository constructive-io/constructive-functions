# Local Development & Debugging

## Prerequisites

- Docker Desktop (or equivalent container runtime)
- Node.js 22+, pnpm 10+

For k8s mode additionally:
- kubectl
- Skaffold (`brew install skaffold` or from https://skaffold.dev)
- kind (only for full k8s cluster locally)

## Two Development Modes

### Mode 1: Docker Compose + Local Node (fastest iteration)

Best for: rapid handler development, no k8s needed.

```bash
# Start infrastructure (postgres, mailpit, graphql-server)
make dev

# Start all functions as local Node processes
make dev-fn
# Or start just one function
node --experimental-strip-types scripts/dev.ts --only=simple-email
```

Functions run as local Node processes with hot-reload via `tsx --watch`. Infrastructure runs in Docker containers.

**Stop:** `make dev-down` (infrastructure) + Ctrl+C (functions)

### Mode 2: Skaffold K8s (production-like)

Best for: testing k8s manifests, job-service integration, e2e tests.

```bash
# Single function (infra + just that function)
make skaffold-dev-simple-email
# or directly:
skaffold dev -p simple-email

# All functions
make skaffold-dev
# or:
skaffold dev -p local-simple

# Knative mode (requires operators: cd k8s && make operators-knative-only)
make skaffold-dev-knative
```

Skaffold builds a single Docker image, deploys to a local k8s cluster, and sets up port-forwards.

## Per-Function Skaffold Profiles

Each function has its own Skaffold profile that deploys only the shared infrastructure + that single function. This saves resources and speeds up the dev loop.

| Profile | What it deploys | Command |
|---------|----------------|---------|
| `simple-email` | infra + simple-email | `skaffold dev -p simple-email` |
| `send-email-link` | infra + send-email-link | `skaffold dev -p send-email-link` |
| `local-simple` | infra + all functions | `skaffold dev -p local-simple` |
| `local` | Knative + all functions | `skaffold dev -p local` |

Profiles are auto-generated by `pnpm generate`. Adding a new function automatically creates its profile.

## Port Map

Ports are defined in each function's `handler.json` (`port` field). Check `generated/functions-manifest.json` for the full mapping.

| Service | Local Port | Source |
|---------|------------|--------|
| Job Service | 8080 | fixed |
| PostgreSQL | 5432 | fixed |
| Constructive Server | 3002 | fixed |
| simple-email | 8081 | handler.json |
| send-email-link | 8082 | handler.json |

## Hot Reload

### Docker Compose mode
Functions run via `tsx --watch` — editing `functions/<name>/handler.ts` triggers an instant restart.

### Skaffold mode
Skaffold syncs `functions/**/*.ts` files into the running container. The container runs `tsx --watch` which detects the change and restarts (~2-5s).

**What triggers a sync:** Any `.ts` file change under `functions/`.
**What triggers a full rebuild:** Changes to `package.json`, `scripts/`, `templates/`, or anything outside `functions/`.

## Debugging

### View pod status

```bash
kubectl get pods -n constructive-functions
```

All pods should be `Running` (except `constructive-db` which completes and shows `Completed`).

### View logs

```bash
# Job service (job pickup, dispatch, errors)
kubectl logs -n constructive-functions -l app=knative-job-service -f

# Specific function
kubectl logs -n constructive-functions -l app=simple-email -f

# Constructive server
kubectl logs -n constructive-functions -l app=constructive-server -f

# DB setup job (ran once at deploy time)
kubectl logs -n constructive-functions -l job-name=constructive-db
```

### Check events (image pull failures, scheduling issues)

```bash
kubectl get events -n constructive-functions --sort-by='.lastTimestamp'
kubectl describe pod <pod-name> -n constructive-functions
```

### Query the database directly

```bash
# Via kubectl exec
kubectl exec -it -n constructive-functions deploy/postgres -- psql -U postgres -d constructive

# Or via port-forward (already active if using Skaffold)
psql -h localhost -p 5432 -U postgres -d constructive
```

Useful queries:

```sql
-- Check pending jobs
SELECT id, task_identifier, attempts, locked_by, last_error
FROM app_jobs.jobs ORDER BY id;

-- Check database record
SELECT id, name FROM metaschema_public.database;

-- Manually insert a test job
SELECT * FROM app_jobs.add_job(
  (SELECT id FROM metaschema_public.database LIMIT 1),
  'simple-email'::text,
  '{"to":"test@example.com","subject":"test","html":"<p>hello</p>"}'::json
);
```

### Exec into a pod

```bash
kubectl exec -it -n constructive-functions deploy/simple-email -- sh
kubectl exec -it -n constructive-functions deploy/knative-job-service -- sh
```

### Restart a deployment

```bash
kubectl rollout restart -n constructive-functions deploy/knative-job-service
kubectl rollout restart -n constructive-functions deploy/simple-email
```

## Common Issues

### ImagePullBackOff for ghcr.io images

The namespace needs a pull secret:

```bash
kubectl create secret docker-registry ghcr-pull \
  --docker-server=ghcr.io \
  --docker-username=<username> \
  --docker-password=<PAT> \
  --docker-email=<email> \
  -n constructive-functions

kubectl patch serviceaccount default -n constructive-functions \
  -p '{"imagePullSecrets": [{"name": "ghcr-pull"}]}'
```

### CrashLoopBackOff

Check logs: `kubectl logs -n constructive-functions deploy/<name> --previous`

Common causes:
- Missing env vars (check ConfigMaps and Secrets)
- Build not run (`pnpm build` before `skaffold dev`)
- Database not ready (wait for `constructive-db` job to complete)

### Port conflicts

If a local port is already in use, Skaffold will log a warning. Check: `lsof -i :<port>`. Either kill the conflicting process or change the port in `handler.json` and re-run `pnpm generate`.

### DB setup job stuck or failed

```bash
kubectl logs -n constructive-functions -l job-name=constructive-db
kubectl delete job constructive-db -n constructive-functions
# Re-deploy to recreate it
```

## Running CI K8s Tests Locally

To reproduce the CI e2e matrix locally:

```bash
# 1. Start the full stack
make skaffold-dev

# 2. In another terminal, run a specific function's e2e test
PGHOST=localhost PGPORT=5432 PGUSER=postgres PGPASSWORD='postgres123!' PGDATABASE=constructive \
  pnpm jest tests/e2e/__tests__/simple-email.e2e.test.ts tests/e2e/__tests__/job-queue.test.ts

# Or run all e2e tests
pnpm test:e2e
```

For single-function isolation (matches CI behavior):

```bash
# Deploy only simple-email
skaffold dev -p simple-email

# Run only its tests
pnpm jest tests/e2e/__tests__/simple-email.e2e.test.ts tests/e2e/__tests__/job-queue.test.ts
```
